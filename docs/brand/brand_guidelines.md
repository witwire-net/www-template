# www-template ブランドガイドライン

> v2.0 — 2026.03 — www-template Inc.

このドキュメントは、www-template Inc. のブランドアイデンティティを一貫して運用するためのガイドラインです。ロゴ、カラー、タイポグラフィ、デザイン原則など、すべてのタッチポイントで統一された体験を届けるための基盤となります。

- **事業内容**: 次世代ソーシャルメディア基盤「www-template」の開発及び運営（BaaSプラットフォーム運営）、および次世代ソーシャルメディアの事業開発・運用支援（SI）
- **Vision**: なんかすごいUX
- **Mission**: コミュニティにテクノロジーで新たな体験を創出する

## 02. ロゴ

www-template ロゴは「アイコンマーク」と「ワードマーク（Noto Sans JP Bold）」の組み合わせで構成されます。横組みでは、左にアイコン・右にワードマークを配置します。

### ロゴロックアップ — 背景別

- **ライト背景**: ブランドカラーのアイコン + ダークテキストのワードマーク
- **ダーク背景**: ブランドカラーのアイコン + ホワイトテキストのワードマーク
- **ブランドカラー背景**: N-950 のアイコン + N-950 テキストのワードマーク

### クリアスペース & 最小サイズ

- **クリアスペース**: アイコンの高さの 25% を最小余白として、ロゴの上下左右に確保してください。
- **最小サイズ**: ロゴロックアップの横幅は 120px 以上で使用してください。アイコン単体は 24px 以上。

## 05. デザイン原則

www-template のすべてのプロダクトとコミュニケーションは、以下の原則に従ってデザインされます。

### 01. 配置がすべて

UIの本質はドットではなく配置。コンポーネントの位置関係、余白のリズム、視線の流れに徹底的にこだわる。対称に逃げるな。非対称配置、オーバーラップ、対角線フローを意識的に使え。50/50の均等分割はデフォルトではなく例外。

### 02. 角を取れ

シンプルで角がないデザイン。丸みのあるコーナー、柔らかなトランジション、フラットなサーフェス。角丸はサイズに意図を持て — sm(0.5rem) はインタラクティブ要素、md(1rem) はカード、lg(1.5rem) は大型コンテナ、full(9999px) はアバターとピル。迷ったら md。

### 03. 文字は読まれない

テキストによる表現は最小限に。アイコン、レイアウト、ビジュアルで伝えられることを文字に頼るな。

### 04. 弱く見えるぞ

難しい表現、長い表現を使うな。わかりやすく、簡潔で、面白く、齟齬のない表現のみ許可される。機知に富んだ表現を。ネットミーム大歓迎。

### 05. 入力か、出力か

UIが促すのはインプットか、アウトプットか。まずそれを明確にせよ。インプットのみ（LP等）ならスクロール連動演出を必須とし、ヒーローには1つの「記憶に残る視覚要素」を置け。Three.js・SVGアニメーション・動画を活用する。アウトプット（ダッシュボード等）なら機能的配置を優先。

### 06. Flat、But Not Empty

ソリッドなサーフェス、クリーンなボーダー、明るいトーン。シャドウとグラスモーフィズムは禁止。ただし「フラット＝ベタ塗り」ではない。ノイズテクスチャ、ドットグリッド、グラデーションメッシュで空間に奥行きを与えよ。色とタイポグラフィと質感で階層を表現する。

### 07. 雰囲気を作れ

すべての画面に「空気」を作れ。ベタ塗りだけの背景は禁止。微細なドットグリッド（1px / 24px間隔）、ノイズオーバーレイ（opacity 0.03〜0.08）、Brand → transparent のフェード帯 — 最低1つの背景テクスチャを含めること。ただしシャドウ・グロー・グラスモーフィズムは引き続き禁止。

### 08. ジェネリックを殺せ

どこかで見たデザインは不合格。実装前に「これは記憶に残るか？」と自問せよ。テンプレート的な配置パターン、均等分割レイアウト、汎用アイコン+汎用テキストの並びに安住するな。1画面に最低1つの「これは www-template だ」と認識できる視覚的特徴を入れろ。

## 07. 写真・画像・テクスチャ

写真・画像は、ブランドの「明るく、クリーンで、現代的」なトーンを反映してください。

- **トーン & ライティング**: 自然光ベースで明るいトーンを基本とする。コントラストは中程度に保ち、影が強すぎる写真は避ける。
- **カラーアクセント**: 可能な場合、ブランドカラー（#5fd9db）系のアクセントを構図内に含める。強制ではないが意識する。
- **被写体**: 人物写真は多様性を反映する。テクノロジー・コミュニティ・コラボレーションのイメージを優先。
- **加工ルール**: ヴィンテージフィルター・極端な彩度変更・HDR加工は禁止。ホワイトバランス調整と軽微な露出補正のみ許可。

### SVG・イラスト

| 種別                                       | 判定   | 備考                                     |
| ------------------------------------------ | ------ | ---------------------------------------- |
| 幾何学パターン（ドット、ライン、グリッド） | ✓ 推奨 | ブランドの「テック感」を支える           |
| ラインアート（アイコン的イラスト）         | ✓ 許可 | Tabler Icons のストローク（1.5）と揃える |
| 手書き風・オーガニックイラスト             | ✗ 禁止 | ブランドトーンと合わない                 |
| 3Dレンダリング・アイソメトリック           | ✗ 禁止 | Flat原則に違反                           |

### 背景テクスチャ標準

原則07「雰囲気を作れ」を実現するための標準テクスチャ。CSS のみで実装し、画像ファイルは使わない。

| テクスチャ         | 実装                                | パラメータ                              |
| ------------------ | ----------------------------------- | --------------------------------------- |
| ドットグリッド     | CSS `radial-gradient`               | 1px ドット / 24px 間隔 / opacity 0.15   |
| ノイズオーバーレイ | SVG `<feTurbulence>` をインライン化 | baseFrequency 0.65 / opacity 0.03〜0.08 |
| ブランドフェード帯 | CSS `linear-gradient`               | Brand 500 → transparent / 高さ 2〜4px   |

### 禁止テクスチャ

| テクスチャ                             | 理由                                 |
| -------------------------------------- | ------------------------------------ |
| チェッカーパターン                     | レトロ感が出てブランドトーンから逸脱 |
| 太ストライプ                           | 視覚的ノイズが強すぎる               |
| 複雑な幾何学模様（タイル、モザイク等） | テクスチャではなく装飾になる         |

## 08. 空間構成

スペーシングトークンの「値」ではなく「使い方」を定義する。余白はデザインの一部であり、埋めるべき空洞ではない。

- **非対称レイアウトを基本にせよ**: 50/50の均等分割はデフォルトではなく例外。60/40、70/30 を基本比率とし、視線を片側に導け。2カラムレイアウトにおいて均等分割を使う場合は意図を明記すること。
- **グリッドブレイク要素**: 1ページに最低1つ、グリッドを無視して画面幅いっぱいに広がる要素（フルブリード）を置け。コンテンツカラムの幅に閉じこもらない瞬間がリズムを生む。
- **余白の意図**: セクション間の最小余白は `3xl (5rem)`。重要セクション（ヒーロー、CTA）の前後は `8rem` 以上を確保。余白を減らす行為は明確な根拠を伴うこと。
- **オーバーラップ**: テキスト × テキストのオーバーラップは禁止（可読性を損なう）。装飾要素（セクション番号、幾何学パターン、アイコン）× コンテンツのオーバーラップは推奨。奥行きと非対称性を生む手法として活用せよ。

## 10. モーション

すべてのアニメーションは意味を持つこと。装飾的な動きは排除し、状態変化やフィードバックを伝えるモーションのみ使用します。

### トランジションプリセット

| 名前   | 値                | 用途                                     |
| ------ | ----------------- | ---------------------------------------- |
| Fast   | 150ms ease-out    | ホバー・フォーカス等の即時フィードバック |
| Normal | 300ms ease-in-out | テーマ切替・パネル開閉                   |
| Bounce | 500ms spring      | 成功フィードバック・注目誘導             |
| Smooth | 320ms ease-out    | コンテナ展開・スライド                   |

### ページロード演出

| ルール               | 仕様                                                                                               |
| -------------------- | -------------------------------------------------------------------------------------------------- |
| ヒーローコンテンツ   | スタガードリビール: 要素ごとに `animation-delay` をずらしてフェードイン。総演出時間 **800ms 以内** |
| Below the fold       | `IntersectionObserver` で `fade-in + translateY(24px)` を発火。`rootMargin: '-10%'`                |
| リスト・グリッド要素 | 各アイテムに `animation-delay: calc(index * 60ms)` でスタガー。最大遅延 **480ms**                  |

### インタラクション規定

| 要素           | ホバー/アクティブ                   | デュレーション |
| -------------- | ----------------------------------- | -------------- |
| ボタン         | `scale(1.02)` + 背景色変化          | 150ms (Fast)   |
| カード         | ボーダー色が Brand に変化           | 150ms (Fast)   |
| テキストリンク | 下線がスライドイン（`scaleX(0→1)`） | 200ms          |
| アイコンボタン | 背景のサークルがフェードイン        | 150ms (Fast)   |

### 禁止するモーション

| 種別                                                     | 理由                                           |
| -------------------------------------------------------- | ---------------------------------------------- |
| 無限ループアニメーション（ローディング以外）             | 注意散漫・アクセシビリティ上の問題             |
| 3D回転・パースペクティブ変形                             | Flat原則に違反                                 |
| 1秒超のトランジション                                    | UIの応答性を損なう                             |
| パララックススクロール                                   | Flat原則と矛盾する奥行き表現                   |
| バウンス・弾性を伴うUI要素の移動（Bounceプリセット以外） | フィードバック以外の場面で使うと安っぽく見える |

## 11. 差別化チェックリスト

デザインレビュー・実装前のセルフチェックとして使え。全項目 Yes でなければリリースするな。

1. **このレイアウトは他のサービスで見たことがあるか？**
   あるなら変えろ。テンプレートの匂いがしたら不合格。

2. **フォントペアリングに個性はあるか？**
   M PLUS 1（ディスプレイ）+ Noto Sans JP（ボディ）の使い分けが機能しているか。見出しとボディが同じ印象になっていないか。

3. **ブランドカラーは「差し色」として効いているか？**
   #5fd9db が面積12%以下で、視線を導く機能を果たしているか。塗りたくっていないか。

4. **記憶に残る視覚要素が1つあるか？**
   ユーザーが5秒後に「あのページ、○○があった」と言える要素。なければ追加せよ。

5. **背景はベタ塗りになっていないか？**
   ドットグリッド、ノイズ、グラデーション帯のうち最低1つが含まれているか確認。

6. **50/50の均等分割に逃げていないか？**
   2カラム構成で均等分割にしている場合、それが最善の選択か意図を問え。

7. **スクロール連動の演出はあるか？（インプットUIの場合）**
   LP・マーケティングページでは IntersectionObserver によるリビールを必須とする。

## 12. 禁止事項（Don'ts）

ブランドの一貫性を守るために、以下の行為を禁止します。

### ロゴの使用例

- ✓ **正しい使い方**: ブランドカラーのアイコン + ダークテキストのワードマーク
- ✗ **禁止**: ロゴの回転・色変更
- ✗ **禁止**: ロゴへのシャドウ・グロー追加
- ✗ **禁止**: 定義外フォントでのワードマーク表示

### 禁止事項一覧

| カテゴリ       | 禁止事項                                                                                                 |
| -------------- | -------------------------------------------------------------------------------------------------------- |
| ロゴ           | 変形（縦横比変更）、回転、不透明度変更、要素の追加・削除                                                 |
| カラー         | ハードコードされた色値の直接使用（トークンを使うこと）                                                   |
| タイポグラフィ | M PLUS 1 / Noto Sans JP / IBM Plex Mono 以外のフォント使用。見出しとボディに同一書体・同一ウェイトの適用 |
